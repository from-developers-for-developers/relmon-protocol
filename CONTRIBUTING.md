# Contributing to RelMon Protocol

This project defines a deterministic, interoperable protocol for monetary data. Contributions can include:

- Reporting inconsistencies
- Suggesting changes/improvements
- Improving documentation

## Reporting Issues

- Check if the issue already exists before opening a new one.
- Provide a **clear description** and expected vs actual behavior.
- Include example payloads if applicable.

## Suggesting Changes

- Open a discussion or issue describing the feature.
- Explain **why itâ€™s useful** and how it interacts with the existing specification/design.
- If possible, provide a small example or pseudo-spec.

## Submitting Changes

1. Fork the repository
2. Create a branch: `git checkout -b new-branch-name`
3. Make your changes
4. Respect the protocol versioning scheme (if any BC changes introduced, increment the major version)
5. Submit a pull request with a clear description of the change

## Code of Conduct

Please follow the [Contributor Covenant](https://www.contributor-covenant.org/) to maintain a respectful community.
